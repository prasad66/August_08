apply() vs call() vs bind() !!!

apply():

The apply() method calls a function with a given this value, and arguments provided as an array.

Syntax

object.objectMethod.apply( objectInstance, arrayOfArguments )

=> objectInstance -> the value for 'this' is passed
=> arrayOfArguments -> if any input is required, passed here in the form of array

Return value

The result of calling the function with the specified this value and arguments.


call():

The apply() method calls a function with a given this value, and arguments provided as individual parameters.

Syntax

object.objectMethod.call( objectInstance, arguments )

=> objectInstance -> the value for 'this' is passed
=> arrayOfArguments -> if any input is required, passed here in the form of individual parameters

Return value

The result of calling the function with the specified this value and arguments.

Difference between call() and apply() method: The only difference is call() method takes the arguments separated by comma while apply() method takes the array of arguments.

bind():

Both call() and apply() are one-time use methodsâ€”if you call the method with the this context it will have it, but the original function will remain unchanged. Sometimes, you might need to use a method over and over with the this context of another object, and in that case you could use the bind method to create a brand new function with an explicitly bound this.

The bind() method creates a new function that, when called, has its this keyword set to the provided value, with a given sequence of arguments preceding any provided when the new function is called.

Syntax

object.objectMethod.bind( objectInstance, arguments )

=> objectInstance -> the value for 'this' is passed
=> arrayOfArguments -> if any input is required, passed here in the form of array

Return value

Bind returns a copy of the function with the supplied this and the arguments.
